# Codebase Intelligence — VISION

## North Star
> Make every codebase **fully understandable** — to any developer or agent, instantly, with grounded confidence.

Code is the largest, least-navigable knowledge base most companies own. The vision is a system that understands code the way a senior engineer who's been there ten years does — and shares that understanding instantly with anyone (human or AI) who needs it.

## 10-year picture
As AI writes more code, *understanding* code becomes the bottleneck — for review, refactoring, debugging, onboarding, and safe change. The team that builds the best **code-understanding engine** owns critical infrastructure for the AI-coding era. Codebase Intelligence aims to be that engine: the canonical, queryable, always-current model of any codebase, embedded in IDEs, agents, CI, and platforms.

## Why it becomes more than a search box
1. **Understanding compounds:** the symbol graph + history + docs become a living model, not a snapshot.
2. **It's the substrate for higher products:** ContextOS (memory/governance), System Design Assistant (design grounded in real code), agent reliability (agents that actually know the code).
3. **Quality is a moat:** RAG-over-code is genuinely hard; most rivals are mediocre. Excellence here is durable.

## Strategic phases
| Horizon | What it is |
|---------|-----------|
| Year 1 | Best-in-class codebase Q&A + search + auto-docs (standalone + API/MCP) |
| Year 2 | Impact analysis, architecture maps, multi-repo org graph, agents |
| Year 3 | Enterprise on-prem; the code-understanding layer other tools build on |
| Year 5+ | The default "model of your code" for humans and AI |

## Principles
1. **Grounded or silent** — cite or say "I don't know"; never bluff about code.
2. **Always current** — incremental reindex on every push; tied to commit SHA.
3. **API/MCP-first** — be the engine others embed, not just an app.
4. **Privacy-first** — code never trains any model (D-010).
5. **Depth over breadth** — better answers beat more features.

## What we won't do
- Be a shallow "chat with repo" wrapper — depth (AST, graph, rerank, evals) is the point.
- Train models on customer code (D-001/D-010).
- Become an IDE — integrate with the ones developers use.

## The bet
That the winner in AI code tooling will be whoever **understands code best**, and that this understanding is a reusable engine powering an entire product family. Owning it is owning the hard part everyone else needs.
