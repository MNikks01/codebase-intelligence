# Codebase Intelligence — SALES

PLG-led; founder-led for early Team/Enterprise + API deals.

## First customer
- The **public OSS-repo demo** converts: someone asks about a famous repo, is impressed, connects their own. Recruit a friendly team/agency from your build-in-public following.
- Free design partner in exchange for feedback + onboarding-time-saved metric + testimonial.
- **Goal:** one team/agency saying "this cut our onboarding/code-understanding time."

## First 10 customers
- Convert design partners → Team as ROI shows.
- Target agencies (clearest ROI: ramp time = cost) and growing teams with gnarly codebases.
- Founder-run onboarding calls → learn objections, fix activation.
- Pursue 1–2 API/agent-builder design partners (strategic distribution).
- **Goal:** 10 paying logos + 1–2 API users; identify the repeatable aha.

## First 100 customers
- Scale the public demo + best content + clearest ICP.
- Self-serve Team checkout for SMB; light mid-market outbound; queue enterprise (no SSO yet).
- API/MCP listing drives developer self-serve.
- Referral loops (devs share accurate tools).
- **Goal:** repeatable Team + growing API revenue + enterprise pipeline.

## First $1K MRR
- ~50 Pro OR ~5–8 Team OR a mix incl. API usage. Reachable from design partners + demo funnel.
- Focus on activation + week-4 retention; publish a case study (onboarding hours saved).

## First $10K MRR (hire trigger)
- ~10–15 Team accounts + API revenue + maybe a small enterprise pilot.
- Hire #1 (full-stack/AI engineer) to reclaim build time; start real enterprise motion (SSO/on-prem now matter).
- Expansion (more repos/seats/queries) is the cheapest growth.

## Sales assets
- Public OSS-repo demo (the proof).
- **Published eval scores** (accuracy claims you can defend — rare and powerful).
- Case studies: onboarding time saved, refactor confidence, AI-answer accuracy.
- Security one-pager (D-010, on-prem) — closes the code-trust objection.
- API quickstart + SDK docs (for ICP-4).

## Objection handling
| Objection | Response |
|-----------|----------|
| "We tried chat-with-repo; it was wrong." | We're AST + graph + rerank + evals; here are published accuracy scores; try the demo. |
| "What about our code's security?" | Never trains a model (D-010); tenant-isolated; secret-redacted; on-prem option. |
| "Sourcegraph is too heavy / Cursor has @codebase." | Lighter to adopt than Sourcegraph; shared + API/MCP unlike per-user editor index. |
| "Price." | ROI = onboarding weeks saved + safer refactors; usually >> cost. |
| "We can build it." | RAG-over-code is hard; buy the engine via API and ship faster. |

## Motion
Self-serve Free→Pro→Team (SMB) + usage-based API (builders); founder-led Team/Enterprise/API until ~$25–50K MRR, then a sales hire. See [HIRING.md](./HIRING.md).
