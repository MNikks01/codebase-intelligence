# Codebase Intelligence — GTM

Developer-led, build-in-public, open-core PLG (portfolio motion — BUSINESS_STRATEGY.md §4).

## Positioning
> **"Understand any codebase in minutes."** Accurate, cited answers about your code — for humans and agents. Deeper than editor search, lighter than enterprise platforms.

Category: code-understanding engine (API/MCP-first), not another IDE.

## Channels (ranked)
1. **Open-source** — the AST chunker + tree-sitter grammars + eval harness → stars, trust, contributors. Devs trust code-tools that are partly open (D-007).
2. **Build in public** — RAG-over-code learnings (chunking, graph RAG, evals) make great technical content + authority.
3. **Content/SEO** — "RAG over code done right," "why your codebase chatbot is wrong," "AST chunking," "impact analysis with a symbol graph." Ranks + converts engineers.
4. **Product Hunt / HN / Reddit** — "connect a repo, get cited answers" demos well.
5. **Free public demo** — let people ask questions about popular OSS repos (no signup) → viral top-of-funnel + proof of accuracy.
6. **YouTube** — onboarding-time-saved demos.
7. **API/MCP distribution** — listed in MCP directories; agents adopt the engine → developers discover the product.
8. **Partnerships** — agencies (onboarding ROI), accelerators, AI-tool ecosystems.

## The "wow" activation
**Time-to-first-cited-answer:** connect repo → useful summary → ask a question → get a correct, cited answer. Optimize onboarding to <10 min. The public demo (ask about a famous OSS repo) delivers the wow with zero signup.

## Launch plan
- Pre-launch: OSS the chunker early for stars; build-in-public the RAG-over-code journey; public OSS-repo demo as a viral asset.
- Private beta: 10–20 design partners (teams + agencies); collect onboarding-time-saved metrics + testimonials; **publish eval scores**.
- Public launch: PH + HN + flagship "RAG over code, done right" post + demo video + design-partner case studies.

## Content pillars
1. RAG-over-code engineering depth (authority/trust).
2. Onboarding & code-understanding ROI (for leads/agencies).
3. Evals & accuracy transparency (differentiator — publish numbers).
4. Founder build-in-public journey.

## Distinct GTM advantage
The **public "ask about any OSS repo" demo** is a uniquely shareable, proof-driven top-of-funnel that competitors with closed/heavy products can't easily match.

## Metrics
Acquisition (stars, demo plays, signups), Activation (time-to-cited-answer), Retention (week-4, queries/user), Revenue (free→paid, API revenue, MRR), Margin (COGS).
